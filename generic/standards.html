Here are defined standards for logging development.

Logging must be done on the HTML. Development must be done on the JSON.

Logging on HTML.

All logging must be made on log.html. Logging contains information:

  • Date of development. YY/MM/DD.

  • Date of log. YY/MM/DD.

  • Type of log. This can be:

    • Cr: creation;
    • De: deletion;
    • Cv: conversion;
    • Dv: diversion; or
    • Ch: direct change;

    that involves:

    • W: a word;
    • Wm: the meaning of a word;
    • Wu: the usage of a word;
    • Expr: an expression; or
    • G: a grammar rule.
    • Lf: a language feature.

    Thus the deletion of a grammar rule would be noted by DeG.

  • Target(s). What was the actual word / meaning / usage / expression / grammar rule created / deleted / converged into something else / diverged into multiples / changed?

  • Results. Fully describe this creation / deletion / conversion / diversion / direct change.

If multiple types of development are involved, they must be logged separately.
